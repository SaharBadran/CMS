High Level 

MVF_Deeplinks_Highlevel_Architecture

 

CMS

Here is the link for Deeplink JSO.

Guide Lines for Deeplink CMS Content:

  1. Screens that are in the side menu should have the same id as MenuCMS side item id. 
  2. Mapped schemes could be duplicated for different screens in the Deeplink CMS.
  3. Mapped schemes will not include query parameters in the schemes. 
  4. Mapped schemes will include the old supported schemes, mcy:// & mvf://. 

Note: Once side menu item is added in menu config and deeplinks config, the app will automatically should work without special handling. 

Actions for current content of menu config:

  1. Change in tablet section the id "bill" to "overview"
  2. Change in tablet section the id "vertrag" to "tariff".
